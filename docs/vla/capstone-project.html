02. Capstone Project

Real-World Application of Physical AI Principles​

This chapter outlines the Capstone Project, designed to provide hands-on experience in applying the principles of Physical AI and Humanoid Robotics to a real-world problem. It details the project requirements, milestones, and expected deliverables, encouraging students to integrate concepts from embodied intelligence, ROS 2, simulation, Isaac SDK, and visual-language-action models.

Project Overview​

The capstone project is your opportunity to synthesize everything learned throughout this course into a functional robotic system. You will design, implement, and demonstrate a physical AI system that addresses a real-world challenge.

Learning Objectives​

  • Integration: Combine multiple technologies (ROS 2, simulation, AI/ML, sensors)
  • System Design: Create modular, maintainable robotics software
  • Problem Solving: Debug and optimize complex systems
  • Documentation: Communicate technical work effectively
  • Evaluation: Test and validate system performance

Timeline​

  • Week 1-2: Project selection, requirements, initial design
  • Week 3-5: Core development and simulation testing
  • Week 6-7: Integration, refinement, real-world testing
  • Week 8: Final demonstration, documentation, presentation

Project Requirements​

Minimum Requirements​

1. Multi-Component Integration

  • At least 3 of the following: ROS 2, Simulation (Gazebo/Isaac/Unity), Sensors, Actuators, AI/ML, Voice/Vision

2. Autonomous Behavior

  • System must demonstrate autonomous decision-making
  • Not purely teleoperated (though teleoperation can be included)

3. Simulation

  • Must be tested in simulation before hardware deployment
  • Document sim-to-real transfer challenges

4. Documentation

  • System architecture diagram
  • Setup/installation instructions
  • Code documentation
  • Performance evaluation

5. Demonstration

  • 5-10 minute video demonstration
  • Live demo or pre-recorded with narration

Suggested Project Ideas​

1. Voice-Controlled Mobile Manipulator​

Description: Robot that responds to voice commands to navigate and manipulate objects in an environment.

Components:

  • Mobile base (differential drive or omnidirectional)
  • Robotic arm (e.g., 6-DOF manipulator)
  • Cameras and depth sensors
  • Whisper speech recognition
  • LLM for command understanding
  • Object detection (YOLO, Faster R-CNN)

Tasks:

  • "Go to the kitchen"
  • "Pick up the red cup"
  • "Place it on the table"

Technical Challenges:

  • Navigation + manipulation coordination
  • Voice-to-action grounding
  • Object detection and pose estimation

ROS 2 Packages Needed:

  • nav2 for navigation
  • moveit2 for motion planning
  • Custom voice control node
  • Vision processing pipeline

Example Architecture:

Voice Input → Whisper → LLM → Task Planner
                                    ↓
                        Navigation Node ← SLAM
                                    ↓
                        Manipulation Node ← MoveIt2
                                    ↓
                        Low-Level Controllers

2. Autonomous Warehouse Assistant​

Description: Robot autonomously navigates warehouse, identifies items, and delivers them to designated locations.

Components:

  • Mobile robot platform
  • 2D/3D LiDAR
  • Camera for barcode/QR scanning
  • Multi-floor navigation capability

Tasks:

  • Receive pickup request (item ID, location)
  • Navigate to shelf
  • Identify and verify item
  • Transport to delivery zone
  • Confirm delivery

Technical Challenges:

  • Dynamic obstacle avoidance
  • Multi-goal path planning
  • Localization accuracy
  • Human-robot collaboration

Evaluation Metrics:

  • Success rate
  • Average delivery time
  • Navigation efficiency
  • Safety (collisions avoided)

3. Social Robot Companion​

Description: Robot that interacts with humans through conversation, facial expressions, and gestures.

Components:

  • Humanoid or social robot platform
  • Camera for face detection
  • Microphone array
  • Expressive actuators (face, arms)
  • Emotion recognition

Tasks:

  • Greet visitors
  • Answer questions about environment
  • Guide people to locations
  • Respond to emotional cues

Technical Challenges:

  • Natural conversation flow
  • Multimodal perception
  • Appropriate behavioral responses
  • Person tracking and re-identification

AI/ML Components:

  • Face detection and recognition
  • Emotion classification
  • LLM for dialogue
  • Text-to-speech synthesis

4. Reinforcement Learning for Locomotion​

Description: Train a legged robot (simulated or real) to walk, navigate obstacles using RL.

Components:

  • Quadruped or bipedal robot
  • Isaac Gym for parallel training
  • Joint position/torque sensors
  • IMU for balance

Tasks:

  • Stable walking on flat terrain
  • Navigate uneven surfaces
  • Climb stairs or slopes
  • Recover from perturbations

Technical Challenges:

  • Reward function design
  • Sim-to-real transfer
  • Stability and safety
  • Energy efficiency

Training Setup:

  • Isaac Gym with 2048+ parallel environments
  • PPO or SAC algorithm
  • Domain randomization
  • Progressive curriculum

5. Vision-Language Navigation​

Description: Robot navigates to locations described in natural language using visual landmarks.

Components:

  • Mobile platform
  • RGB-D camera
  • LiDAR for mapping
  • Vision-language model

Example Commands:

  • "Go to the room with the blue chair"
  • "Navigate to the door next to the potted plant"
  • "Find the hallway with paintings on the wall"

Technical Challenges:

  • Semantic scene understanding
  • Language grounding to visual features
  • Long-horizon planning
  • Visual place recognition

ML Components:

  • CLIP for vision-language matching
  • Semantic segmentation
  • Object detection
  • Scene graph generation

System Architecture Template​

Layered Architecture​

┌─────────────────────────────────────────┐
│         Application Layer               │
│  (Task planning, decision making)       │
└─────────────────────────────────────────┘
                  ↕
┌─────────────────────────────────────────┐
│         Perception Layer                │
│  (Sensor processing, SLAM, detection)   │
└─────────────────────────────────────────┘
                  ↕
┌─────────────────────────────────────────┐
│         Planning Layer                  │
│  (Motion planning, trajectory gen)      │
└─────────────────────────────────────────┘
                  ↕
┌─────────────────────────────────────────┐
│         Control Layer                   │
│  (PID, model-predictive, RL policy)     │
└─────────────────────────────────────────┘
                  ↕
┌─────────────────────────────────────────┐
│         Hardware Layer                  │
│  (Motors, sensors, communication)       │
└─────────────────────────────────────────┘

ROS 2 Node Structure​

Perception Nodes:

  • camera_node: Publishes images
  • lidar_node: Publishes laser scans
  • slam_node: Publishes odometry and map
  • detector_node: Publishes detected objects

Planning Nodes:

  • task_planner: High-level task sequencing
  • path_planner: Global path planning
  • motion_planner: Trajectory generation

Control Nodes:

  • navigation_controller: Velocity commands
  • arm_controller: Joint commands
  • gripper_controller: Grasp commands

Interface Nodes:

  • voice_interface: Speech input
  • visualization: RViz markers
  • safety_monitor: Emergency stop

Implementation Roadmap​

Phase 1: Design (Week 1-2)​

Deliverables:

  • Project proposal (2-3 pages)
  • System architecture diagram
  • Component list and justification
  • Risk analysis

Tasks:

  1. Define problem and requirements
  2. Research existing solutions
  3. Design system architecture
  4. Identify hardware/software needs
  5. Create development timeline

Phase 2: Simulation (Week 3-4)​

Deliverables:

  • Working simulation environment
  • Core functionality demonstrated
  • Baseline performance metrics

Tasks:

  1. Set up simulation (Gazebo/Isaac Sim/Unity)
  2. Create robot model (URDF/SDF)
  3. Implement sensor simulation
  4. Develop core algorithms
  5. Test in varied scenarios

Example: Gazebo Setup

# Create workspace
mkdir -p ~/capstone_ws/src
cd ~/capstone_ws/src

# Create package
ros2 pkg create my_capstone --build-type ament_python \
    --dependencies rclpy geometry_msgs sensor_msgs nav2_msgs

# Build
cd ~/capstone_ws
colcon build
source install/setup.bash

Phase 3: Integration (Week 5-6)​

Deliverables:

  • Integrated system with all components
  • Documented interfaces
  • Integration test results

Tasks:

  1. Integrate perception pipeline
  2. Connect planning and control
  3. Add safety mechanisms
  4. Implement error handling
  5. Conduct system testing

Integration Checklist:

  • All ROS 2 nodes communicate correctly
  • Sensor data flows to processing nodes
  • Commands reach actuators
  • Emergency stop functional
  • Logging and debugging enabled

Phase 4: Optimization (Week 7)​

Deliverables:

  • Performance benchmark results
  • Optimization report
  • Refined system

Tasks:

  1. Profile system performance
  2. Optimize computational bottlenecks
  3. Tune control parameters
  4. Improve robustness
  5. Conduct stress testing

Optimization Areas:

  • Latency: Reduce sensor-to-action delay
  • Throughput: Increase processing rate
  • Accuracy: Improve detection/localization
  • Reliability: Handle edge cases
  • Efficiency: Reduce resource usage

Phase 5: Documentation & Demo (Week 8)​

Deliverables:

  • Final report (10-15 pages)
  • Code repository (GitHub)
  • Demonstration video
  • Presentation slides

Tasks:

  1. Write comprehensive documentation
  2. Record demonstration video
  3. Prepare presentation
  4. Submit code and artifacts
  5. Present to class/panel

Evaluation Criteria​

Technical Implementation (40%)​

  • Functionality: Does it work as specified?
  • Integration: Are components well-integrated?
  • Code Quality: Clean, modular, documented?
  • Robustness: Handles errors gracefully?
  • Performance: Meets performance requirements?

Innovation & Complexity (20%)​

  • Novelty: Original approach or application?
  • Difficulty: Appropriate technical challenge?
  • AI/ML Integration: Effective use of learning?
  • Multi-modal: Combines multiple modalities?

Documentation (20%)​

  • Clarity: Easy to understand?
  • Completeness: All aspects covered?
  • Reproducibility: Can others replicate?
  • Visuals: Diagrams, charts, screenshots?

Demonstration (20%)​

  • Presentation: Clear communication?
  • Video Quality: Professional production?
  • Live Demo: Successful execution?
  • Q&A: Understands technical details?

Testing Strategy​

Unit Tests: Test individual components

import unittest
from my_capstone.planning import PathPlanner

class TestPathPlanner(unittest.TestCase):
    def test_straight_line(self):
        planner = PathPlanner()
        path = planner.plan(start=(0,0), goal=(1,0))
        self.assertAlmostEqual(len(path), 2)

if __name__ == '__main__':
    unittest.main()

    pass

System Tests: End-to-end validation

  • Run full system in simulation
  • Execute representative scenarios
  • Measure success rates

Common Pitfalls​

  1. Scope Creep: Start simple, add features iteratively
  2. Late Integration: Integrate early and often
  3. Ignoring Edge Cases: Test failure modes
  4. Poor Documentation: Document as you develop
  5. Overfitting to Sim: Test sim-to-real transfer early
  6. Neglecting Safety: Implement emergency stops
  7. Last-Minute Demos: Practice demonstrations early

Resources​

Hardware Recommendations​

  • Budget (<$500): Raspberry Pi + Arduino + cheap sensors
  • Mid-Range ($500-2000): Jetson Nano + better sensors + servos
  • Research ($2000+): Jetson Orin + LiDAR + vision + arm

Simulation Platforms​

  • Gazebo: Best for traditional ROS 2 integration
  • Isaac Sim: Best for AI/ML, photorealistic rendering
  • Unity: Best for HRI, VR/AR

Useful Libraries​

  • Navigation: nav2, slam_toolbox
  • Manipulation: moveit2, pinocchio
  • Vision: opencv, pytorch, detectron2
  • Voice: whisper, pyttsx3, speech_recognition

Final Checklist​

Before submission, ensure:

  • Code runs without errors
  • All dependencies documented
  • README with setup instructions
  • Architecture diagram included
  • Performance metrics reported
  • Video demonstration uploaded
  • Presentation slides prepared
  • GitHub repository public
  • All team members credited
  • License file added

Example Project Report Structure​

  1. Abstract (1 paragraph)
  2. Introduction (1-2 pages)
    • Problem statement
    • Motivation
    • Objectives
  3. Related Work (1-2 pages)
    • Literature review
    • Existing solutions
  4. System Design (2-3 pages)
    • Architecture
    • Components
    • Interfaces
  5. Implementation (3-4 pages)
    • Algorithms
    • Code structure
    • Key challenges
  6. Evaluation (2-3 pages)
    • Metrics
    • Results
    • Analysis
  7. Conclusion (1 page)
    • Summary
    • Limitations
    • Future work
  8. References
  9. Appendices (code listings, extra figures)

Key Takeaways​

  • Start simple, iterate rapidly
  • Test early and often
  • Document throughout development
  • Integrate components incrementally
  • Plan for demo failures
  • Communicate progress regularly
  • Learn from challenges
  • Have fun building!
