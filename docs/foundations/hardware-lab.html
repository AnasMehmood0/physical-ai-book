02. Hardware Lab

Sim Rig vs Edge Kit specs​

This chapter delves into the specifications and comparative analysis of two primary hardware setups for physical AI development: the Simulation Rig (Sim Rig) and the Edge Kit. It outlines the technical details, computational capabilities, sensor suites, and actuator types typically found in each.

Overview​

Physical AI development requires different hardware configurations depending on your goals:

  • Simulation Rig: High-performance workstation for training and testing in virtual environments
  • Edge Kit: Embedded computing platform for deploying AI on physical robots
  • Hybrid Setup: Combination approach for complete development workflow

Understanding the trade-offs between these approaches is essential for efficient Physical AI development.

Simulation Development Rig​

Purpose and Use Cases​

A simulation rig is optimized for:

  • Training AI models: Deep learning, reinforcement learning
  • Physics simulation: Gazebo, Isaac Sim, MuJoCo, Unity
  • Large-scale testing: Parallel simulations, Monte Carlo testing
  • Development iteration: Fast code-test-debug cycles

Hardware Specifications​

GPU (Critical Component)​

Recommended:

  • NVIDIA RTX 4090 (24GB VRAM) - Best performance
  • NVIDIA RTX 4080 (16GB VRAM) - Good balance
  • NVIDIA RTX 3090 (24GB VRAM) - Budget option with high VRAM

Minimum:

  • NVIDIA RTX 3060 (12GB VRAM)
  • NVIDIA GTX 1080 Ti (11GB VRAM)

Why NVIDIA?

  • CUDA support required for PyTorch, TensorFlow
  • Isaac Sim requires RTX GPUs with ray tracing
  • Better driver support for robotics software

VRAM Considerations:

  • 8GB: Basic simulations, small models
  • 12GB: Medium models, single robot simulations
  • 16-24GB: Large models, multi-robot simulations, Isaac Sim

CPU​

Recommended:

  • AMD Ryzen 9 7950X (16 cores)
  • Intel Core i9-13900K (24 cores)
  • AMD Threadripper (for parallel simulations)

Minimum:

  • 8 cores, 16 threads
  • Base clock 3.0+ GHz

Why it matters:

  • Physics simulation is CPU-intensive
  • Parallel environment training (e.g., Isaac Gym)
  • Compilation of large codebases

RAM​

Recommended:

  • 64GB DDR5
  • 32GB minimum for serious work

Breakdown:

  • 16GB: OS and background processes
  • 16GB: Simulation engine
  • 16GB: AI training
  • 16GB+: Large datasets, multiple environments

Storage​

Configuration:

  • OS Drive: 500GB NVMe SSD (PCIe 4.0)
  • Data Drive: 2TB NVMe SSD for datasets
  • Backup: 4TB HDD for model checkpoints

Why NVMe?

  • Fast dataset loading (ImageNet, robotic datasets)
  • Quick checkpoint saving during training
  • Faster Docker container operations

Software Stack​

Operating System:

  • Ubuntu 22.04 LTS (recommended for ROS 2)
  • Ubuntu 20.04 LTS (for older ROS 2 versions)
  • Windows 11 (with WSL2 for Linux tools)

Essential Software:

  • ROS 2: Humble Hawksbill or Iron Irwini
  • Simulation: Gazebo Fortress, Isaac Sim, Unity ML-Agents
  • AI Frameworks: PyTorch 2.0+, TensorFlow 2.x
  • Docker: For containerized development
  • NVIDIA Drivers: 525+ with CUDA 11.8 or 12.x

Edge Deployment Kit​

Purpose and Use Cases​

Edge kits are designed for:

  • On-robot inference: Running trained models on physical hardware
  • Real-time control: Low-latency sensor processing and actuation
  • Autonomous operation: Battery-powered, mobile deployment
  • Field testing: Real-world validation

Popular Edge Platforms​

NVIDIA Jetson Family​

Jetson Orin Nano (Recommended for learning):

  • GPU: 1024 CUDA cores, 32 Tensor cores
  • CPU: 6-core ARM Cortex-A78AE
  • RAM: 8GB LPDDR5
  • Power: 7-15W
  • Price: ~$500
  • Use case: Mobile robots, drones, small humanoids

Jetson Orin NX:

  • GPU: 1024 CUDA cores, 32 Tensor cores
  • CPU: 8-core ARM Cortex-A78AE
  • RAM: 16GB LPDDR5
  • Power: 10-25W
  • Price: ~$800
  • Use case: Medium-sized robots, warehouse automation

Jetson AGX Orin:

  • GPU: 2048 CUDA cores, 64 Tensor cores
  • CPU: 12-core ARM Cortex-A78AE
  • RAM: 32-64GB LPDDR5
  • Power: 15-60W
  • Price: ~$2,000
  • Use case: Autonomous vehicles, complex manipulation

Key Advantages:

  • Native CUDA support
  • JetPack SDK with ROS 2 support
  • TensorRT for optimized inference
  • Isaac ROS packages
  • Power-efficient AI acceleration

Raspberry Pi (Budget Option)​

Raspberry Pi 5:

  • CPU: Quad-core ARM Cortex-A76 @ 2.4GHz
  • RAM: 4GB or 8GB LPDDR4
  • GPU: VideoCore VII (no CUDA)
  • Power: 5W
  • Price: ~$80
  • Use case: Simple robots, learning, hobby projects

Limitations:

  • No GPU acceleration for deep learning
  • Lower compute power
  • Best for classical robotics, not heavy AI

Good for:

  • Educational robots
  • Sensor data collection
  • ROS 2 message routing
  • Lightweight vision (OpenCV)

Intel NUC / Mini PCs​

Intel NUC 13 Extreme:

  • CPU: Core i9-13900K (desktop-class)
  • GPU: Can fit RTX 4070 (via PCIe)
  • RAM: Up to 64GB DDR5
  • Power: 650W
  • Price: ~$1,500-2,000
  • Use case: Stationary robots, research platforms

Advantages:

  • Desktop-class performance
  • Upgradeable GPU
  • Standard x86 software compatibility

Disadvantages:

  • Higher power consumption
  • Larger form factor
  • Not battery-friendly

Peripherals and Sensors​

Essential Additions:

  • Camera: Intel RealSense D435i ($300) or generic USB webcam ($50)
  • LiDAR: RPLIDAR A1 ($100) or Hokuyo UST-10LX ($1,500)
  • IMU: BNO055 ($30) or VectorNav VN-100 ($500)
  • Motor Controllers: ODrive ($120), VESC ($150), or Dynamixel servos ($50-300 each)
  • Power: Lithium battery (varies), voltage regulators, power distribution

Hybrid Development Workflow​

Recommended Setup​

For Most Developers:

  1. Sim Rig: Mid-range GPU workstation ($2,000-3,000)
  2. Edge Kit: Jetson Orin Nano ($500) + basic sensors ($500)
  3. Total: ~$3,000-4,000

Workflow:

  1. Develop and train on sim rig (fast iteration)
  2. Export optimized model (TensorRT, ONNX)
  3. Deploy to edge kit for real-world testing
  4. Collect real-world data, transfer back to sim rig
  5. Retrain with real data, redeploy

Cloud Alternative​

When to use cloud:

  • No upfront hardware cost
  • Access to latest GPUs (A100, H100)
  • Scalable for large experiments
  • Team collaboration

Cloud Providers:

  • AWS EC2: g5.xlarge ($1/hour) to p4d.24xlarge ($32/hour)
  • Google Cloud: n1-standard-8 + T4 (~$0.50/hour)
  • Lambda Labs: A100 instances (~$1.10/hour)
  • Paperspace: Gradient platform with Jupyter notebooks

Cost Comparison:

  • Sim rig purchase: $2,000 upfront, $0 ongoing
  • Cloud at 40 hours/week: $0 upfront, ~$2,000/year (at $1/hour)
  • Break-even: ~1 year of heavy usage

Limitations:

  • No physical robot testing
  • Network latency for remote development
  • Ongoing costs can exceed hardware

Software Setup Guide​

Simulation Rig Setup​

Step 1: Install Ubuntu 22.04

# Create bootable USB with Ubuntu 22.04 LTS
# Install with "Minimal installation" option

Step 2: Install NVIDIA Drivers

sudo apt update
sudo apt install nvidia-driver-525
sudo reboot
nvidia-smi  # Verify installation

Step 3: Install ROS 2 Humble

sudo apt install software-properties-common
sudo add-apt-repository universe
sudo apt update && sudo apt install curl -y
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key -o /usr/share/keyrings/ros-archive-keyring.gpg

echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] http://packages.ros.org/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null

sudo apt update
sudo apt install ros-humble-desktop

Step 4: Install PyTorch

pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118

Step 5: Install Gazebo

sudo apt install ros-humble-gazebo-ros-pkgs

Jetson Edge Kit Setup​

Step 1: Flash JetPack

  • Download NVIDIA SDK Manager
  • Flash JetPack 5.1+ to Jetson device
  • Includes Ubuntu 20.04, CUDA, cuDNN, TensorRT

Step 2: Install ROS 2

# On Jetson (ARM architecture)
sudo apt update
sudo apt install ros-humble-ros-base

Step 3: Install Isaac ROS

sudo apt install ros-humble-isaac-ros-common

Step 4: Optimize for inference

Validation and Testing​

Sim Rig Validation​

GPU Test:

# Run PyTorch CUDA test
python -c "import torch; print(torch.cuda.is_available())"
python -c "import torch; print(torch.cuda.get_device_name(0))"

Gazebo Test:

ros2 launch gazebo_ros gazebo.launch.py
# Should open Gazebo GUI without errors

Training Test:

# Run small training loop
import torch
model = torch.nn.Linear(10, 1).cuda()
optimizer = torch.optim.Adam(model.parameters())
for i in range(100):
    x = torch.randn(32, 10).cuda()
    y = model(x).mean()
    y.backward()
    optimizer.step()
print("Training successful!")

Edge Kit Validation​

Inference Test:

import torch
model = torch.jit.load("model.pt")

Common Issues and Solutions​

Sim Rig Issues​

Issue: CUDA out of memory

  • Solution: Reduce batch size, use gradient accumulation, enable mixed precision training

Issue: Gazebo crashes

  • Solution: Update drivers, reduce physics simulation rate, use headless mode

Issue: Slow training

  • Solution: Check GPU utilization (nvidia-smi), use data loading workers, optimize data pipeline

Edge Kit Issues​

Issue: Model too slow on Jetson

  • Solution: Quantize model (INT8), use TensorRT, reduce model size, prune unnecessary layers

Issue: Out of memory on Jetson

  • Solution: Reduce input resolution, use model pruning, enable swap memory, close background processes

Issue: Power consumption too high

  • Solution: Use Jetson power modes (nvpmodel), optimize code, reduce inference frequency

Key Takeaways​

  • Sim Rig: High-performance GPU workstation for training and simulation
  • Edge Kit: Power-efficient embedded platform for on-robot deployment
  • NVIDIA Jetson: Best edge platform for AI robotics (CUDA support)
  • Hybrid workflow: Develop in simulation, deploy to edge
  • Budget: $1,250-$3,900 for sim rig, $400-$2,000 for edge kit
  • Cloud: Alternative to sim rig for scalable training
  • Validation: Test GPU, simulation, training pipeline before starting development
