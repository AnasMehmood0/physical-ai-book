# Quickstart: Initial Docusaurus Setup & Chapter Structure

This guide provides instructions to quickly set up and verify the Docusaurus project for the "Physical AI and Humanoid Robotics (Interactive Textbook)".

## Prerequisites

- Node.js (LTS version recommended)
- npm or Yarn package manager

## 1. Initialize Docusaurus Project

Navigate to the root of your repository and run the following command to create the Docusaurus project in the `web/` directory:

```bash
npx create-docusaurus@latest web classic --typescript
```

This command will:
- Create a new directory named `web/`.
- Initialize a Docusaurus project within it using the `classic` template.
- Configure it for TypeScript.

## 2. Install Dependencies

Navigate into the `web/` directory and install the project dependencies:

```bash
cd web
npm install # or yarn install
```

## 3. Configure Sidebar Structure

Edit `web/sidebars.js` to define the 5 chapter entries. Replace the existing `tutorialSidebar` content with the following:

```javascript
/**
 * Creating a sidebar enables you to:
 - Create an ordered group of docs
 - Render a sidebar in the docs sidebars.
 - Add custom categories (e.g. Intro, Chapter 1, etc.)
 */

const sidebars = {
  // By default, Docusaurus generates a sidebar from the docs folder structure
  // tutorialSidebar: [{type: 'autogenerated', dirName: '.'}],

  // Create a sidebar manually
  tutorialSidebar: [
    'intro',
    {
      type: 'category',
      label: 'Chapter 1: Introduction to Physical AI',
      link: {type: 'doc', id: 'chapter1'},
      items: []
    },
    {
      type: 'category',
      label: 'Chapter 2: Sensors & Actuators',
      link: {type: 'doc', id: 'chapter2'},
      items: []
    },
    {
      type: 'category',
      label: 'Chapter 3: Humanoid Locomotion',
      link: {type: 'doc', id: 'chapter3'},
      items: []
    },
    {
      type: 'category',
```

## 4. Create Chapter Markdown Files

Create empty markdown files for each chapter in `web/docs/`. For example:

- `web/docs/chapter1.md`
- `web/docs/chapter2.md`
- `web/docs/chapter3.md`
- `web/docs/chapter4.md`
- `web/docs/chapter5.md`

Add basic front matter to each file, e.g., for `chapter1.md`:

```markdown
---
id: chapter1
title: Introduction to Physical AI
---

## Welcome to Chapter 1!

This is the beginning of our journey into Physical AI.
```

## 5. Configure for Custom React Components in Markdown

This is typically supported by default in Docusaurus v2+ when using MDX. To explicitly ensure this, verify `web/docusaurus.config.js` has `remarkPlugins` or `rehypePlugins` configured if custom MDX processing is ever needed. For basic React component embedding, no explicit configuration is usually needed beyond ensuring `.mdx` files can render React.

Ensure `docusaurus.config.js` is set to use MDX for markdown files (which is the default for `classic` preset).

## 6. Apply Academic Color Scheme

Edit `web/src/css/custom.css`. You can modify the CSS variables to achieve a modern, clean, academic color scheme. For example:

```css
/**
 * Any CSS here will be applied to the entire Docusaurus site.
 */

/* For example, a red theme: */
:root {
  --ifm-color-primary: #212121;       /* Dark Grey */
  --ifm-color-primary-dark: #1a1a1a;
  --ifm-color-primary-darker: #171717;
  --ifm-color-primary-darkest: #111111;
  --ifm-color-primary-light: #333333;
  --ifm-color-primary-lighter: #404040;
  --ifm-color-primary-lightest: #555555;
  --ifm-code-font-size: 95%;
  --docusaurus-highlighted-code-line-bg: rgba(0, 0, 0, 0.1);
}

/* Specific adjustments for academic feel */
html[data-theme='dark'] {
  --ifm-background-color: #1a1a1a;
  --ifm-font-color-base: #e0e0e0;
  --ifm-heading-color: #f5f5f5;
}

html[data-theme='light'] {
  --ifm-background-color: #ffffff;
  --ifm-font-color-base: #333333;
  --ifm-heading-color: #212121;
}

/* Example of subtle border for code blocks */
.docusaurus-highlighted-code-line {
  border-left: 3px solid var(--ifm-color-primary);
  padding-left: 10px;
}

/* Adjust typography for academic readability */
body {
  font-family: 'Georgia', serif; /* Or 'Times New Roman' */
  line-height: 1.6;
}

h1, h2, h3, h4, h5, h6 {
  font-family: 'Palatino Linotype', 'Book Antiqua', Palatino, serif;
}

/* Links for academic references */
a {
  color: #007bff; /* A standard academic blue */
}

a:hover {
  color: #0056b3;
}
```

This CSS snippet provides a dark grey primary theme for dark mode, a clean white for light mode, and adjusts font families for an academic feel.

## 7. Run Docusaurus Development Server

From the `web/` directory, start the development server:

```bash
npm start # or yarn start
```

Open `http://localhost:3000` in your browser to verify:
- The Docusaurus site loads successfully.
- The sidebar displays all 5 chapters with correct titles.
- The applied color scheme is visible.
- (Future verification): A simple custom React component embedded in a markdown file renders correctly.
